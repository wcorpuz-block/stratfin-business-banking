# Legacy Forecasting Model Analysis - Interview Template

## Stakeholder Interview Guide for Model Discovery

Since legacy models may not be locally accessible, use these interview templates to gather information about existing forecasting structures, drivers, and outputs.

---

## Finance Team Interview Template

### Meeting Objective
Understand current forecasting model structure, drivers, and outputs to ensure consistency with new business banking framework.

### Pre-Meeting Preparation
- [ ] Schedule 60-minute meeting with FP&A lead
- [ ] Request sample model files if possible
- [ ] Prepare to take detailed notes on structure and methodology

### Interview Questions

#### A. Current Forecasting Framework
1. **"Can you walk me through our current forecasting process?"**
   - How often do we update forecasts?
   - What's the typical forecast horizon?
   - Who are the key stakeholders for forecast outputs?

2. **"What are the main models we use for financial planning?"**
   - Quarterly business reviews
   - Annual budget planning
   - Product-specific forecasts
   - Customer growth projections

3. **"How is our forecasting structured by business unit?"**
   - Square POS business
   - Payment processing
   - Lending products
   - Any existing banking products

#### B. Model Drivers and Assumptions
4. **"What are the primary drivers in our revenue forecasts?"**
   - Customer acquisition rates
   - Transaction volumes
   - Average transaction values
   - Pricing/fee structures
   - Market growth assumptions

5. **"How do we model costs in our forecasts?"**
   - Fixed vs. variable cost allocation
   - Headcount planning methodology
   - Technology and infrastructure costs
   - Processing and operational costs

6. **"What external assumptions do we typically use?"**
   - Market growth rates
   - Economic indicators
   - Competitive factors
   - Regulatory changes

#### C. Model Structure and Calculations
7. **"Can you describe the typical structure of our financial models?"**
   - Input assumptions sheet
   - Calculation engine
   - Output dashboards
   - Scenario analysis capabilities

8. **"How do we calculate unit economics?"**
   - Revenue per customer
   - Cost per transaction
   - Customer lifetime value
   - Payback periods

9. **"What's our approach to scenario planning?"**
   - Base, upside, downside cases
   - Sensitivity analysis methods
   - Key variables for scenario testing

#### D. Outputs and Reporting
10. **"What are the key outputs from our forecasting models?"**
    - P&L projections
    - Cash flow forecasts
    - Key performance indicators
    - Executive dashboards

11. **"How do we present forecast results to leadership?"**
    - Monthly/quarterly formats
    - Variance analysis approach
    - Key metrics highlighted
    - Scenario comparisons

12. **"What reporting cadence do we follow?"**
    - Monthly actuals vs. forecast
    - Quarterly reforecasts
    - Annual planning cycle
    - Ad-hoc analysis requests

#### E. Model Quality and Validation
13. **"How do we validate our forecast accuracy?"**
    - Historical accuracy tracking
    - Assumption validation process
    - Model back-testing approach
    - Continuous improvement methods

14. **"What are the current limitations or challenges with our models?"**
    - Data availability issues
    - Model complexity concerns
    - Update time requirements
    - Stakeholder feedback

#### F. Business Banking Specific
15. **"Do we have any existing models for banking products?"**
    - Payment processing forecasts
    - Card program projections
    - Transfer/ACH volume models
    - Customer adoption curves

16. **"How should new business banking models integrate with existing frameworks?"**
    - Consistency requirements
    - Reporting alignment needs
    - Stakeholder expectations
    - Update process integration

## Follow-Up Actions Template

### Immediate Next Steps (Post-Interview)
- [ ] Request access to sample model files
- [ ] Schedule follow-up with model owners
- [ ] Document key findings and requirements
- [ ] Compare with business banking framework

### Model Access Requests
- [ ] SharePoint/network drive permissions
- [ ] Historical model versions
- [ ] Template files and documentation
- [ ] Assumption validation sources

### Validation Activities
- [ ] Review model accuracy against actuals
- [ ] Test key assumption sensitivity
- [ ] Compare methodologies with industry best practices
- [ ] Identify improvement opportunities

### Integration Planning
- [ ] Align business banking drivers with existing framework
- [ ] Standardize calculation methodologies
- [ ] Ensure consistent output formats
- [ ] Plan unified reporting approach

## Success Metrics for Legacy Analysis

### Information Gathering
- [ ] Complete understanding of current forecasting process
- [ ] Documentation of all key model drivers
- [ ] Mapping of calculation structures and flows
- [ ] Identification of standard outputs and formats

### Quality Assessment
- [ ] Historical accuracy validation
- [ ] Assumption reasonableness review
- [ ] Model limitation identification
- [ ] Best practice documentation

### Integration Readiness
- [ ] Alignment plan with existing frameworks
- [ ] Consistent methodology approach
- [ ] Unified reporting structure
- [ ] Stakeholder requirement clarity

This interview-based approach will help you gather the legacy model information you need even if the actual files aren't immediately accessible.
