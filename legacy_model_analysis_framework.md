# Legacy Forecasting Model Analysis Framework

## Model Discovery and Documentation Process

### Step 1: Model Inventory
**Objective:** Locate and catalog all existing forecasting models

#### Common Model Locations to Check:
- [ ] Local drives (C:\Users\[username]\Documents)
- [ ] Network drives (shared finance folders)
- [ ] SharePoint/OneDrive locations
- [ ] Email attachments from prior periods
- [ ] Finance team shared folders

#### Model Types to Look For:
- [ ] Annual budget models
- [ ] Quarterly forecast updates
- [ ] Product-specific P&L models
- [ ] Customer acquisition models
- [ ] Revenue forecasting models
- [ ] Cost allocation models
- [ ] Scenario planning models

### Step 2: Model Structure Analysis
**For each model identified, document:**

#### A. Model Overview
- **Model Name/Version:**
- **Last Updated:**
- **Created By:**
- **Purpose/Scope:**
- **Forecast Period:**
- **Update Frequency:**

#### B. Input Drivers (Key Variables)
- **Volume Drivers:**
  - Customer counts
  - Transaction volumes
  - Usage rates
  - Market size assumptions
  
- **Pricing Drivers:**
  - Fee structures
  - Rate assumptions
  - Pricing changes over time
  
- **Cost Drivers:**
  - Fixed costs
  - Variable costs
  - Headcount assumptions
  - Technology costs
  
- **Market Drivers:**
  - Growth rates
  - Market share assumptions
  - Competitive factors
  - Economic assumptions

#### C. Calculation Structure
- **Revenue Calculations:**
  - How revenue is built up
  - Key formulas used
  - Dependencies between variables
  
- **Cost Calculations:**
  - Cost allocation methods
  - Driver-based vs. fixed costs
  - Shared cost allocation
  
- **Profitability Metrics:**
  - Gross margin calculations
  - Operating profit methodology
  - Unit economics approach

#### D. Model Outputs
- **Financial Statements:**
  - P&L structure
  - Balance sheet items
  - Cash flow projections
  
- **Key Performance Indicators:**
  - Revenue metrics
  - Customer metrics
  - Operational metrics
  - Profitability ratios
  
- **Reporting Formats:**
  - Monthly/quarterly views
  - Product breakdowns
  - Variance analysis
  - Scenario comparisons

### Step 3: Model Quality Assessment

#### Technical Quality
- [ ] Formula accuracy and consistency
- [ ] Data validation and error checking
- [ ] Version control and documentation
- [ ] Scenario and sensitivity analysis capability

#### Business Logic
- [ ] Assumption reasonableness
- [ ] Driver relationships make sense
- [ ] Outputs align with business reality
- [ ] Historical accuracy validation

#### Usability
- [ ] Clear structure and navigation
- [ ] Adequate documentation
- [ ] Easy to update and maintain
- [ ] Appropriate level of detail

### Step 4: Comparison with New Framework

#### Methodology Alignment
- [ ] Similar driver approaches
- [ ] Consistent calculation methods
- [ ] Compatible assumption frameworks
- [ ] Aligned output formats

#### Gap Analysis
- [ ] Missing drivers in legacy models
- [ ] Additional complexity needed
- [ ] Improved accuracy opportunities
- [ ] Enhanced scenario capabilities

## Questions to Ask About Each Model

### Strategic Questions
1. What business decisions does this model support?
2. Who are the primary users and stakeholders?
3. How often is the model updated and reviewed?
4. What level of accuracy has been achieved historically?

### Technical Questions
1. What are the key assumptions and how are they validated?
2. How sensitive are outputs to changes in key drivers?
3. What scenario analysis capabilities exist?
4. How is the model maintained and version controlled?

### Process Questions
1. What data sources feed into the model?
2. How long does it take to update the model?
3. What quality assurance processes are in place?
4. How are results communicated to stakeholders?
